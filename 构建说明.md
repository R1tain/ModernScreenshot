# 构建说明

## 环境要求

本项目需要在 **Windows** 环境下编译，因为：
1. 使用了 Windows Forms（仅 Windows 支持）
2. 调用了 Windows API（user32.dll）
3. 使用了 GDI+ 图形库（Windows 特有）

## 安装 .NET SDK

### Windows 10/11

1. 访问 https://dotnet.microsoft.com/download
2. 下载 .NET 6.0 SDK（或更高版本）
3. 运行安装程序
4. 验证安装：
   ```cmd
   dotnet --version
   ```

## 编译步骤

### 方法 1：使用构建脚本（推荐）

```cmd
cd ModernScreenshot
build.bat
```

构建脚本会自动完成：
- 清理旧文件
- 还原 NuGet 包
- 编译项目
- 发布单文件可执行程序

输出位置：
```
bin\Release\net6.0-windows\win-x64\publish\ModernScreenshot.exe
```

### 方法 2：手动编译

```cmd
cd ModernScreenshot

# 还原依赖
dotnet restore

# 编译（Debug）
dotnet build

# 编译（Release）
dotnet build -c Release

# 运行
dotnet run

# 发布独立版本
dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true
```

## 常见问题

### Q: 编译报错 "找不到 Windows Forms"
A: 确保使用的是 .NET 6.0 或更高版本，且 csproj 中有：
```xml
<UseWindowsForms>true</UseWindowsForms>
```

### Q: 运行时提示 "需要 .NET 运行时"
A: 使用 `--self-contained` 参数发布独立版本，或安装 .NET 6.0 运行时。

### Q: 能在 Linux/Mac 上编译吗？
A: 不能。Windows Forms 和 Windows API 仅支持 Windows 平台。

## 发布选项

### 单文件发布（推荐）
```cmd
dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

优点：
- 单个 exe 文件
- 无需安装 .NET 运行时
- 约 50-80 MB

### 依赖框架发布（体积小）
```cmd
dotnet publish -c Release -r win-x64 --no-self-contained
```

优点：
- 文件约 200 KB
- 需要用户安装 .NET 6.0 运行时
