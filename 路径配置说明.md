# 路径配置说明

## 概述

ModernScreenshot 所有数据都存储在安装目录下，默认安装到 **D:\ModernScreenshot**，不会在 C 盘留下任何文件。

---

## 目录结构

```
D:\ModernScreenshot\                 # 应用程序根目录
├── ModernScreenshot.exe             # 主程序
├── *.dll                            # 依赖库
├── Screenshots\                     # 截图保存目录
│   ├── 截图_20260613_143025.png
│   ├── 截图_20260613_143156.png
│   └── ...
├── Config\                          # 配置文件目录
│   └── settings.json                # （预留）
└── Temp\                            # 临时文件目录
    └── （临时文件）
```

---

## 路径说明

### 1. 应用程序根目录
- **默认位置**: `D:\ModernScreenshot`
- **可自定义**: 安装时可选择其他位置
- **自动创建**: 首次运行时自动创建子目录

### 2. 截图保存目录
- **路径**: `{安装目录}\Screenshots`
- **用途**: 保存所有截图文件
- **命名规则**: `截图_yyyyMMdd_HHmmss.png`
- **示例**: `截图_20260613_143025.png`

### 3. 配置文件目录
- **路径**: `{安装目录}\Config`
- **用途**: 保存应用程序配置（预留功能）
- **文件**: `settings.json`（未来版本使用）

### 4. 临时文件目录
- **路径**: `{安装目录}\Temp`
- **用途**: 临时文件缓存
- **自动清理**: 可通过代码定期清理

---

## 核心功能

### PathManager 类

负责统一管理所有路径：

```csharp
// 获取应用程序根目录
string appDir = PathManager.AppDirectory;
// 输出: D:\ModernScreenshot

// 获取截图目录
string screenshotsDir = PathManager.ScreenshotsDirectory;
// 输出: D:\ModernScreenshot\Screenshots

// 获取配置目录
string configDir = PathManager.ConfigDirectory;
// 输出: D:\ModernScreenshot\Config

// 获取默认截图保存路径
string path = PathManager.GetDefaultScreenshotPath();
// 输出: D:\ModernScreenshot\Screenshots\截图_20260613_143025.png
```

## 路径选择逻辑

### 1. 优先使用 D 盘
```csharp
// 如果应用程序在 C 盘运行
if (_appDirectory.StartsWith("C:", StringComparison.OrdinalIgnoreCase))
{
    // 且 D 盘存在
    if (Directory.Exists("D:\\"))
    {
        // 自动切换到 D:\ModernScreenshot
        _appDirectory = @"D:\ModernScreenshot";
    }
}
```

### 2. 回退机制
- 如果 D 盘不存在，使用应用程序所在目录
- 保证在任何情况下都能正常运行

## 安装程序配置

使用 **Inno Setup** 创建安装程序（setup.iss）:

### 默认安装位置
```ini
DefaultDirName=D:\ModernScreenshot
```

### 安装时检查
```pascal
function InitializeSetup(): Boolean;
begin
  if not DirExists('D:\') then
    MsgBox('警告: D 盘不存在，将使用默认安装路径。', mbInformation, MB_OK);
end;
```

### 自动创建目录
```pascal
procedure CurStepChanged(CurStep: TSetupStep);
begin
  if CurStep = ssPostInstall then
  begin
    CreateDir(ExpandConstant('{app}\Screenshots'));
    CreateDir(ExpandConstant('{app}\Config'));
    CreateDir(ExpandConstant('{app}\Temp'));
  end;
end;
```

## 保存文件行为

### 截图保存
```csharp
var saveDialog = new SaveFileDialog
{
    Filter = "PNG图片|*.png|JPEG图片|*.jpg",
    FileName = $"截图_{DateTime.Now:yyyyMMdd_HHmmss}.png",
    InitialDirectory = PathManager.ScreenshotsDirectory  // 默认打开截图目录
};
```

### 用户体验
1. 按 Ctrl+S 保存截图
2. 文件对话框自动打开到 `D:\ModernScreenshot\Screenshots`
3. 文件名自动生成（带时间戳）
4. 保存后显示完整路径通知

## 卸载行为

### 自动清理
```ini
[UninstallDelete]
Type: filesandordirs; Name: "{app}\Screenshots"
Type: filesandordirs; Name: "{app}\Config"
Type: filesandordirs; Name: "{app}\Temp"
```

### 卸载时清理
- 删除应用程序目录
- 删除所有子目录和文件
- 删除注册表项（如果有）
- **不留任何残留文件**

## 使用示例

### 场景1: 普通用户
1. 运行安装程序
2. 默认安装到 D:\ModernScreenshot
3. 所有截图保存在 D:\ModernScreenshot\Screenshots
4. 卸载时自动清理

### 场景2: 绿色版用户
1. 编译后复制整个 bin 目录到 D:\ModernScreenshot
2. 直接运行 ModernScreenshot.exe
3. 自动创建 Screenshots、Config、Temp 目录
4. 无需安装，删除目录即可清理

### 场景3: 多用户环境
1. 每个用户可以安装到不同目录
2. 例如: D:\ModernScreenshot_User1, D:\ModernScreenshot_User2
3. 互不干扰，数据独立

## 构建和打包

### 1. 编译发布版本
```bash
dotnet publish -c Release -r win-x64 --self-contained false
```

### 2. 生成安装程序
```bash
# 使用 Inno Setup 编译器
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" setup.iss
```

### 3. 输出
- 安装程序: `ModernScreenshot_Setup.exe`
- 大小: 约 5-10 MB（取决于依赖项）

---

## 注意事项

1. **D 盘空间**: 建议至少 100 MB 可用空间
2. **权限要求**: 普通用户权限即可（不需要管理员）
3. **.NET 运行时**: 需要 .NET 6.0 Desktop Runtime
4. **自动清理**: 可定期清理 Temp 目录
