# 构建和打包指南

## 前置要求

### 软件依赖
1. **.NET 6.0 SDK** - 用于编译项目
2. **Inno Setup 6** - 用于生成安装程序
   - 下载地址: https://jrsoftware.org/isdl.php

---

## 构建步骤

### 1. 编译发布版本

#### 方式1: 自包含版本（推荐）
```bash
# 包含 .NET 运行时，用户无需安装 .NET
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=false
```

**优点**: 用户无需安装 .NET 运行时  
**缺点**: 体积较大（约 70 MB）

#### 方式2: 依赖框架版本
```bash
# 需要用户安装 .NET 6.0 Desktop Runtime
dotnet publish -c Release -r win-x64 --self-contained false
```

**优点**: 体积小（约 1 MB）  
**缺点**: 需要用户预装 .NET 6.0 Desktop Runtime

#### 使用现有脚本
```bash
# Windows
build.bat

# Linux/Mac (交叉编译)
./build.sh
```

---

### 2. 验证编译结果

编译完成后，检查输出目录：
```
bin\Release\net6.0-windows\win-x64\publish\
├── ModernScreenshot.exe          # 主程序
├── ModernScreenshot.dll          # 程序集
├── *.dll                         # 依赖库
└── ModernScreenshot.runtimeconfig.json
```

---

### 3. 生成安装程序

#### 步骤1: 确认 Inno Setup 已安装
```bash
# 检查 Inno Setup 是否安装
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" /?
```

#### 步骤2: 编译安装脚本
```bash
# 命令行方式
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" setup.iss

# 或使用 GUI 方式
# 1. 打开 Inno Setup Compiler
# 2. File -> Open -> 选择 setup.iss
# 3. Build -> Compile
```

#### 步骤3: 获取安装程序
输出文件: `ModernScreenshot_Setup.exe`  
位置: 项目根目录

---

## 安装程序配置

### setup.iss 关键配置

```ini
[Setup]
DefaultDirName=D:\ModernScreenshot       # 默认安装到 D 盘
PrivilegesRequired=lowest                # 普通用户权限
DisableDirPage=no                        # 允许用户选择安装路径
```

### 安装选项

1. **桌面快捷方式** - 默认不勾选
2. **开机自动启动** - 默认不勾选

用户可在安装时自主选择。

---

## 绿色版打包

如果不需要安装程序，可以直接打包 publish 目录：

```bash
# 1. 复制 publish 目录
xcopy /E /I bin\Release\net6.0-windows\win-x64\publish ModernScreenshot_Portable

# 2. 创建 ZIP 压缩包
# 使用 7-Zip 或其他压缩工具
7z a -tzip ModernScreenshot_Portable.zip ModernScreenshot_Portable\
```

### 绿色版使用说明
1. 解压到任意目录（建议 D 盘）
2. 运行 ModernScreenshot.exe
3. 自动创建 Screenshots、Config、Temp 子目录
4. 无需安装，删除目录即可清理

## 常见问题

### Q: 如果用户没有 D 盘怎么办？
A: PathManager 会自动检测，回退到应用程序所在目录。

### Q: 可以安装到 C 盘吗？
A: 可以，安装时可以自定义路径。

### Q: 卸载会删除截图吗？
A: 是的，卸载会删除整个应用程序目录，建议用户先备份截图。

### Q: 如何更新版本？
A: 直接运行新版本安装程序，选择相同的安装路径覆盖安装。
